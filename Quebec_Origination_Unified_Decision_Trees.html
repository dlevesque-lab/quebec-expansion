Tree U1 · Application Language How LOS auto-sets the Application Language from the Primary Borrower — with Underwriter override & clear

→ from Tree B2: Deal has been onboarded and assigned.

LOS FR-01
Primary Borrower language preference has been set
  1. LOS auto-sets the Application Language from the Primary Borrower's language preference.
  2. The Application Language now tracks the Primary Borrower — this is the default AUTO mode (system-managed).
LOS FR-01
No manual override — LOS manages the Application Language (AUTO)
  1. The Application Language stays linked to the Primary Borrower's language preference.
LOS
Primary Borrower is changed, OR their language preference changes
  1. LOS automatically re-derives the Application Language = the new Primary Borrower's language preference.
  2. Re-evaluated automatically on every such change.
LOS FR-01
No change to the Primary Borrower or preference
  1. The Application Language is unchanged.
  2. It still equals the Primary Borrower's language preference.
LOS
Underwriter manually sets / overrides the Application Language
  1. Underwriter manually sets the Application Language to a chosen value.
  2. Mode switches to MANUAL — the value is locked to the Underwriter's selection.
Underwriter LOS FR-01
  1. The manually-set Application Language persists.
  2. Any change to the Primary Borrower or their language preference does NOT replace it.
  3. The manual value persists permanently.
LOS

  1. The override is cleared.
  2. Application Language resets to the current Primary Borrower's language preference.
  3. Returns to AUTO mode and resumes tracking future Primary Borrower changes.
Underwriter LOS FR-01
Underwriter keeps the override
  1. The manually-set Application Language persists permanently.
  2. It is immune to all Primary Borrower / language-preference changes.

All paths resolve the current Application Language

→ Conditions are generated and displayed in the current Application Language (French or English).

LOS FR-01
AUTO mode (default): The Application Language is derived from — and continuously tracks — the Primary Borrower's language preference. Changing the Primary Borrower, or that borrower's preference, re-derives the Application Language.
MANUAL override: Once an Underwriter sets the Application Language, it is locked and persists permanently. Primary Borrower / language-preference changes no longer affect it.
Clearing the override: An Underwriter can clear the manual setting, which resets the Application Language to the current Primary Borrower preference and returns the system to AUTO mode (resumes automatic tracking).
