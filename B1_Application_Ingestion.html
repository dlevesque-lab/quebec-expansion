Mortgage Broker completes and submits a Quebec mortgage application via BOSS POS platform.

Mortgage Broker BRK-01 FR-08
  1. BIS polls BOSS POS at regular intervals to check for available deals.
  2. BIS determines that a new deal is available for retrieval.
  3. BIS makes an API call to BOSS POS to pull the deal payload (XML or JSON).
  4. BIS stores the raw Broker POS payload in the BIS data store.
BIS Broker POS
  1. BIS transforms the raw Broker POS payload into the LOS schema format.
  2. Quebec-specific: French-accented characters in borrower names, addresses, and property fields must be preserved through transformation (NFR-13). French street types are pre-mapped in LOS.
  3. Quebec-specific: BIS parses French-language credit bureau if submitted in French (NFR-02). Parsing output validated against English equivalent.
  4. Note: BOSS POS schema is identical for Quebec and rest-of-Canada regardless of language setting (AS-04). English schema is always used.
BIS NFR-02 NFR-13
Transformation successful
  1. BIS stores the transformed deal in the BIS transformed deal store.
  2. BIS publishes a Pub/Sub message indicating the deal is received and ready to ingest.
BIS
  1. LOS listens for the Pub/Sub notification and determines when to ingest the deal.
  2. LOS makes an API call to BIS to pull the transformed deal.
  3. LOS onboards the deal and populates application data fields.
LOS FR-01
Subject Property Province NOT Quebec
  1. Application data onboarded.
  2. Follow standard non-Quebec deal flow.
LOS alt path (non-Quebec)
Subject Property Province = Quebec (supported)
  1. Application data onboarded.
  2. LOS sets Application Language Preference based on the Primary Borrower's language preference.
  3. LOS applies Quebec Property Tiering rules to assign the appropriate risk tier to the subject property (NFR-15).
  4. LOS validates Quebec as a supported province via Qualification Checks (NFR-07).
LOS NFR-07 NFR-15
  1. Application data pushed to Salesforce CRM.
  2. Loan appears in Broker Portal.
Salesforce CRM FR-15
  1. BIS listens for Application Status Update events from LOS.
  2. BIS sends an acknowledgement message to Broker POS confirming deal receipt.
  3. Broker receives confirmation in BOSS POS.
BIS Broker POS

→ Ingestion complete. Proceed to Tree B2: Deal Allocation Process.

LOS Deal Allocation Process FR-07 UWR-01
Transformation failed

⚠ Slack alert sent to BIS team notifying of transformation failure.

BIS ⚠ Slack Alert
  1. BIS team investigates and manually fixes the transformation issue.
  2. Deal remains in BIS queue; not ingested until transformation succeeds.
  3. Once fixed, deal re-enters the transformation pipeline from Step 2.
BIS Team
Note: If LOS ingestion fails after a successful transformation, a Slack failure notification is sent to the BIS team and the deal remains in the BIS queue for retry.
Quebec-Specific Requirements Referenced in This Tree
Req ID Type Summary Priority
FR-01 Functional Application-Level Language Preference set at LOS ingestion based on Primary Borrower language; saved to App Intake snapshot and pushed downstream to Salesforce and LMS. Must Have
FR-08 Functional Broker Portal & Kapti must support bilingual French/English display. Language preference captured at onboarding and stored in Salesforce. Must Have
FR-15 Functional Salesforce client-facing email communications must respect the borrower's language preference, captured and stored at deal ingestion. Must Have
NFR-02 Non-Functional LOS/BIS must parse French-language credit bureaus submitted by Quebec brokers. Initial testing via BOSS shows mostly French parsing works; full validation required. Must Have
NFR-07 Non-Functional LOS Qualification Checks must allow Quebec as a supported province. Minimum Lending Value and Subject Property Province checks to be adjusted. No net-new qualification checks. Must Have
NFR-13 Non-Functional All systems (LOS, Broker Portal, Salesforce, DMS) must accept and correctly handle French-accented text (e.g., é, à, ê, ç) in all free-form fields through the ingestion pipeline. Must Have
NFR-15 Non-Functional LOS must automatically calculate Property Tier for Quebec properties at ingestion to assign the appropriate risk score and pricing to the loan. Must Have
Key Assumptions (AS)
AS-03: All client and broker communication defaults to French unless an English preference is explicitly stated.
AS-04: BOSS POS submission schema is identical for Quebec and rest-of-Canada regardless of language setting. Confirmed by M3. French street types are pre-mapped in LOS.
AS-08: Broker onboarding remains identical to rest-of-Canada except: outreach in French first, AMF licence number captured, Quebec Sales Region assigned.
AS-15: When multiple borrowers have differing language preferences, the Primary Borrower's language preference governs the application language.
